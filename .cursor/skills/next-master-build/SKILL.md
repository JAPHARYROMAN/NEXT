# NEXT Master Build

## When to use

Apply this skill at the start of any substantive NEXT work: new features,
services, infrastructure, UX flows, discovery/ranking logic, or refactors
that affect architecture or product direction.

## Instructions

1. Read the full constitution: [constitution.md](constitution.md)
2. Treat it as a **foundational system directive** — not optional guidance.
3. Align every decision with Sections 1–12 (directive, philosophy, architecture,
   products, layers, engineering, UX, discovery, design, stack, performance,
   final directive).
4. If existing repo code diverges from the constitution, follow the repo for
   immediate edits, then note gaps and evolve toward the constitution.
5. Reject patterns explicitly forbidden (MVP/tutorial tone, monoliths,
   engagement hacking, AI-as-controller, tutorial code, hardcoded secrets).

## Quick compliance checklist

- [ ] Modular, event-driven, observable, resilient, zero-trust
- [ ] AI as invisible infrastructure — not behavioral control
- [ ] Unified identity, design language, intelligence, events, emotional tone
- [ ] Layered stack: Experience → Application → Intelligence → Social/Economic → Data/Events → Infra → Edge
- [ ] Production-grade, strongly typed, tested, documented deliverables
- [ ] Discovery: precision + expansion + chaos; no addiction/rage optimization
- [ ] Preferred stack (Next.js, React, TS, Go/Rust, Kafka, K8s, etc.) unless repo dictates otherwise

## Authority

The verbatim constitution in [constitution.md](constitution.md) is the source of truth.
